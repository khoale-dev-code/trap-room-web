# TRAP Room Reservation Page V28

The public reservation page has been rebuilt around a clearer two-column
experience:

- editorial blue visit card;
- address, opening hours and phone details;
- three-step booking explanation;
- polished form card;
- date protection against past dates;
- accessible phone/name autocomplete;
- large guest stepper from 1 to 30;
- note character count;
- clear booking disclaimer;
- 58–60 px mobile action targets;
- responsive Chrome/Safari mobile layout with safe-area padding.

The API payload remains compatible with the existing reservation endpoint:
`customerName`, `phone`, `date`, `time`, `guestCount`, and `note`.
